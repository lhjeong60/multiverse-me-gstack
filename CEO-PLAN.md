## Accepted Scope (added to Phase 1)

### 1. "차원 균열" 로딩 텍스트
EEAAO 패러디 문자열 배열, 연극적 로딩 중 랜덤 표시.
예: "모든 곳에 동시에 존재하는 중...", "빨래방 우주를 지나는 중..."

### 2. "다시 점프" 버튼
결과 페이지에서 verse jump만 재시작, 사진 재업로드 불필요.
- "다시 점프" 클릭 → Verse Jump #1 선택 화면으로 돌아감
- 사진은 세션에 유지, 새 선택 → 새 우주 세트 생성
- **Rate limit에 포함**: "다시 점프"도 1회 생성으로 카운트 (IP당 하루 3회)
- 3회 소진 시: "오늘의 차원 점프를 모두 사용했습니다. 내일 다시 도전하세요"

### 3. "우주 해금" 게이미피케이션
- **상태 저장**: localStorage (서버 비용 0, 기기별 독립)
- **해금 조건**: 같은 사진으로 다른 verse jump 선택 시 새 우주가 해금됨
- **UI**: 결과 그리드에서 해금 안 된 우주는 "?" 실루엣 + "다시 점프로 해금"
- **진행도**: "5/15 우주 발견" 프로그레스 바
- **M effort 근거**: 해금 상태 관리(S) + 해금 진행 UI(S) + "다시 점프"와 연동 로직(S) = M

### 4. 카카오톡 SDK 연동
- **선행 조건**: 카카오 개발자 앱 등록 + 도메인 인증 (Phase 1 시작 전에 해둘 것, 심사 1-3일)
- **공유 템플릿**: Feed 타입 (OG 이미지 1장 + 제목 + 설명)
- **OG 이미지 활용**: @vercel/og로 생성한 단일 대표 이미지를 카카오 공유 이미지로 재활용
- **Fallback**: SDK 로드 실패 시 URL 복사 + 카카오톡 딥링크로 대체
