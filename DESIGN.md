## Problem Statement

이름과 사진을 입력하면 여러 평행세계에서의 내 모습과 스토리를 보여주는 재미용 웹페이지.
영화 "Everything Everywhere All at Once"에서 영감을 받은 프로젝트. 핵심은 단순
결과 나열이 아니라 "차원을 넘나드는" 시네마틱 경험을 웹에서 재현하는 것.

## What Makes This Cool

1. **Verse Jumping 메카닉**: 영화처럼 "통계적으로 개연성 없는 생뚱맞은 행동"을 선택하면
   차원이 열림. 행동의 개연성이 떨어질수록 더 기상천외한 우주에 도착. 1점짜리(신발 바꿔 신기)를
   고르면 현실적인 우주(스타트업 CEO), 3점짜리(모르는 사람에게 프로포즈)를 고르면
   기상천외한 우주(핫도그 손가락 세계)로. 선택에 인과가 있으니 바이럴 + 재방문 가치.
2. **내 얼굴이 반영된 AI 이미지**: Nano Banana를 활용해 사용자 얼굴이 유지된 채
   각 평행세계 테마에 맞는 초상화 생성.
3. **시네마틱 트랜지션**: 글리치, 포탈, 차원 균열 애니메이션으로 세계 간 점프 연출.
   결과를 보는 과정 자체가 영화 같은 경험.
4. **공유 카드**: 5개 우주의 나를 콜라주한 OG 이미지 카드 → SNS에서 "나도 해보고 싶다" 유발.

## Constraints

- 사이드 프로젝트 (1인 개발)
- SNS 바이럴이 목표 → 모바일 퍼스트
- AI 이미지 생성 비용: 사용자당 ~$0.30-1.00 (API 선택에 따라)
- Nano Banana를 이미지 생성 도구로 사용 (사용자가 테스트 완료, 얼굴 유지 확인)
- 이미지 생성 예상 응답 시간: 5~15초/장 (Nano Banana 기준, 확인 필요)

## Privacy & Data Policy

- 업로드된 사진은 이미지 생성 후 서버에서 즉시 삭제 (클라이언트 → API → 삭제)
- 생성된 결과 이미지만 저장 (공유 URL용), 원본 사진은 보관하지 않음
- 결과 저장 기간: 30일 후 자동 삭제 (또는 사용자 수동 삭제)
- 업로드 시 동의 문구: "사진은 이미지 생성에만 사용되며 즉시 삭제됩니다"
- 얼굴 미검출 시: "얼굴을 찾을 수 없습니다. 정면 사진을 올려주세요" 안내

## Premises

1. Nano Banana를 사용해서 사용자 사진 기반 평행세계 이미지를 생성할 수 있다.
2. 시네마틱 트랜지션(글리치/포탈 애니메이션)은 프론트엔드 CSS/JS로 구현 가능하다.
3. SNS 바이럴을 위해 고유 결과 URL + OG 이미지 카드가 필수다.
4. AI 이미지 생성 비용은 사이드 프로젝트 수준에서 감당 가능하다.
5. "verse jumping" 메카닉(황당한 행동 선택)이 바이럴 계수를 높인다.

## Cross-Model Perspective

Claude 서브에이전트의 독립 리뷰:
- **가장 쿨한 미고려 버전**: "분기점 질문"으로 인터랙티브하게 만드는 아이디어.
  → 사용자 피드백으로 한 단계 더 진화: 인생 분기점 대신 영화의 "verse jumping"
  메카닉(황당한 행동 선택)으로 변경.
- **핵심 인사이트**: 사용자가 가장 흥분하는 지점은 이미지 퀄리티가 아닌
  "시네마틱 연출". 트랜지션 연출에 리소스를 집중해야 함.
- **기술 추천**: IP-Adapter/FaceID 기반 도구 → 사용자가 Nano Banana로 대체 결정.
- **주말 빌드 순서**: 트랜지션 연출 먼저, 이미지 퀄리티는 API가 해결.

## Approaches Considered

### Approach A: "Weekend Ship" (Minimal Viable)
- Next.js + Nano Banana + GPT-4o-mini
- 5개 고정 우주, 황당한 선택지 2개, 기본 페이드 트랜지션
- Effort: S (주말) / Risk: Low
- Pros: 빠른 배포, 비용 예측 가능
- Cons: 트랜지션이 단순, 우주 다양성 제한

### Approach B: "Cinematic Full" (Ideal Architecture)
- Next.js + Nano Banana + GPT-4o + Framer Motion + 사운드 디자인
- 황당한 선택지 3개, 동적 우주 생성(8-12개), 글리치/포탈 트랜지션, 앰비언스 사운드
- Effort: M (1-2주) / Risk: Med
- Pros: "와" 팩터 극대화, 영화 같은 경험
- Cons: 프론트엔드 연출 시간 소요, 이미지 대기 시간이 흐름 깨뜨릴 가능성

### Approach C: "Progressive Experience" (선택됨)
- 트랜지션 연출을 최고 수준으로 먼저 만들고, 나머지는 이터레이션
- A를 배포 후 바이럴 데이터 보면서 B 요소 추가
- Effort: S→M (단계적) / Risk: Low
- Pros: 빠른 배포 + 시네마틱 연출의 조합, 실제 반응 보고 방향 결정

### Phase 1 vs Phase 2 경계

| 기능 | Phase 1 (주말) | Phase 2 (반응 보고) |
|------|---------------|-------------------|
| 트랜지션 | 포탈 오픈 + 페이드 + 기본 글리치 | RGB 분리, 차원 분열, 사운드 동기화 |
| 이미지 생성 | Nano Banana 5개 우주 | 동적 우주 8-12개 |
| 스토리 | GPT-4o-mini 3줄 | GPT-4o 상세 내러티브 |
| 사운드 | 없음 | Howler.js 앰비언스 |
| 공유 | OG 이미지 (단일 대표 이미지) | 5개 콜라주 카드, 인스타 9:16 |
| 저장 | Vercel KV (간단) | Supabase (확장성) |

## Recommended Approach

**Approach C: Progressive Experience**

승부처는 "차원이 열리는 순간의 연출". 이미지 퀄리티는 Nano Banana가 해결.
트랜지션에 모든 크리에이티브 에너지를 집중하고, 나머지는 빠르게 배포 후 이터레이션.

### Verse Jumping 메카닉 — 개연성 스코어 시스템

핵심 설정: **행동의 개연성이 떨어질수록 더 기상천외한 우주에 도착한다.**

각 선택지에 "개연성 역전 점수"(absurdity score)가 있음:
- **1점** (약간 이상함): 일상에서 한 번쯤 할 수 있는 행동
- **2점** (상당히 이상함): 사회적으로 안 하지만 불가능하진 않은 행동
- **3점** (통계적 불가능): 어떤 상황에서도 절대 안 할 행동

**Verse Jump #1 선택지:**
| 선택지 | 점수 |
|--------|------|
| "지금 신발 왼오른을 바꿔 신으시겠습니까?" | 1 |
| "카페에서 주문을 노래로 하시겠습니까?" | 2 |
| "길 가다 모르는 사람에게 프로포즈하시겠습니까?" | 3 |

**Verse Jump #2 선택지:**
| 선택지 | 점수 |
|--------|------|
| "비 오는 날 우산 없이 걸으시겠습니까?" | 1 |
| "회의 중에 책상 위로 올라가시겠습니까?" | 2 |
| "지금 당장 공항에 가서 가장 먼저 뜨는 비행기를 타시겠습니까?" | 3 |

### 선택 → 우주 매핑 로직

2회 점프의 점수를 합산 (최소 2점 ~ 최대 6점).
합산 점수가 높을수록 현실에서 멀어진 우주로 이동.

```
합산 점수 → 우주 세트 (현실적 ←→ 기상천외)

2점 (현실 가까움): 스타트업 CEO, 시골 농부, 뉴욕 재즈 뮤지션, 
                   극지 탐험가, 서커스 공중그네
3점:              뉴욕 재즈 뮤지션, 극지 탐험가, 서커스 공중그네,
                   해적선장, 사이버펑크 해커
4점 (중간):       해적선장, 사이버펑크 해커, 궁중 요리사,
                   우주 탐험가, 사무라이
5점:              우주 탐험가, 사무라이, 심해 아틀란티스 주민,
                   차원 사이 떠도는 존재, 시간을 거꾸로 사는 사람
6점 (극단):       차원 사이 떠도는 존재, 시간을 거꾸로 사는 사람,
                   바위로 환생한 나, 핫도그 손가락 세계의 나, 
                   모든 우주의 나를 동시에 사는 존재
```

핵심: 2점은 "다른 직업을 가진 현실적인 나", 6점은 영화처럼
"존재의 형태 자체가 바뀐 나". 스펙트럼의 양 끝이 극적으로 다름.

"핫도그 손가락 세계"는 영화 직접 오마주. 팬이라면 알아보고 웃음.

### 우주 테마 풀 — 현실 ↔ 기상천외 스펙트럼

**Tier 1: 현실적 (점수 2-3)**
1. 스타트업 CEO (실리콘밸리 세계)
2. 시골 농부 (제주도 세계)
3. 뉴욕 재즈 뮤지션 (1960s 재즈 클럽)
4. 극지 탐험가 (남극 세계)
5. 서커스 공중그네 (유랑 서커스단)

**Tier 2: 판타지 (점수 3-5)**
6. 해적선장 (카리브해 세계)
7. 사이버펑크 해커 (네온시티 2089)
8. 궁중 요리사 (조선 세계)
9. 우주 탐험가 (은하연합)
10. 사무라이 (전국시대)

**Tier 3: 기상천외 (점수 5-6)**
11. 심해 아틀란티스 주민 (수중 문명)
12. 차원 사이 떠도는 존재 (형태 없는 의식체)
13. 시간을 거꾸로 사는 사람 (역행 세계)
14. 바위로 환생한 나 (만물이 의식을 가진 세계)
15. 핫도그 손가락 세계의 나 (영화 오마주)

### 우주 생성 방식: 하이브리드

**Tier는 고정, 그 안에서 LLM이 변주.**

합산 점수로 Tier 배분이 결정되면, 각 슬롯에 대해 GPT-4o-mini가 구체적인
우주를 생성. 예시 프롬프트:

```
Tier 1 슬롯: "현실 세계에서 가능한 직업/상황 하나를 발명해줘. 
  구체적인 장소, 시대, 직함 포함. 예시 수준: 제주도 감귤 농부, 
  뉴욕 재즈 클럽 피아니스트. 짧은 세계관 설명 1줄 추가."

Tier 2 슬롯: "역사/판타지에 기반한 모험적 세계 하나를 발명해줘.
  실제 역사나 장르 소설에서 영감 받되, 구체적 설정 추가.
  예시 수준: 카리브해 해적선장, 조선 궁중 요리사."

Tier 3 슬롯: "물리 법칙이나 존재 형태 자체가 다른 기상천외한 세계 
  하나를 발명해줘. 영화 EEAAO 수준의 상상력.
  예시 수준: 핫도그 손가락 세계, 바위로 환생한 세계."
```

**장점:**
- 매번 다른 결과 → 재방문 가치 극대화
- Tier로 품질 범위를 잡아두니 일관성 유지
- 이미지 프롬프트도 LLM이 함께 생성 (세계관에 맞는 구체적 프롬프트)

**Fallback:** LLM 생성 실패 시 각 Tier의 고정 테마(위 목록)에서 랜덤 선택.

V1: Tier별 고정 테마 3개씩 준비 (fallback용) + LLM 동적 생성 시도.
V2: Tier별 고정 테마 5개 + LLM 생성 품질 개선.

### 바이럴 장치

1. **OG 이미지 카드**: 5개 우주 콜라주 + 이름 + "나의 멀티버스" 타이틀
2. **고유 URL**: `/result/[id]` → 링크만 공유하면 친구도 결과 확인 가능
3. **CTA**: 결과 하단 "당신의 평행세계도 확인해보세요" → 메인으로 유도
4. **인스타 스토리 최적화**: 9:16 비율 이미지 카드 다운로드 가능

### 트랜지션 연출 상세

이것이 이 프로젝트의 핵심 차별화.

1. **포탈 오픈** (Landing → Verse Jump)
   - 화면 중앙에서 원형 글리치 확장
   - 배경이 왜곡되며 빨려들어가는 효과
   - 차원 균열 사운드

2. **차원 분열** (Verse Jump 선택 시)
   - 화면이 3방향으로 갈라지며 선택된 방향으로 빨려들어감
   - RGB 분리 글리치 + 화면 떨림
   - "차원 점프 완료" 글리치 텍스트

3. **우주 리빌** (각 우주 등장)
   - 블랙아웃 → 우주 테마 컬러가 번지며 드러남
   - AI 초상화가 글리치하며 서서히 선명해짐
   - 각 우주별 앰비언스 사운드 페이드인

4. **연극적 로딩 + 순차 리빌** (AI 이미지 생성 대기)
   - 5개 이미지를 병렬로 생성 요청
   - 첫 번째 이미지 완성 즉시 Universe #1 리빌 (나머지는 백그라운드 계속 생성)
   - 사용자가 스토리를 읽는 동안 다음 이미지 준비
   - 아직 안 된 이미지는 "차원 스캔 중..." 글리치 연출로 대기
   - 예상 체감 대기: 첫 우주까지 ~8초, 이후 스와이프 시 즉시 or 3-5초

### Error States

| 시나리오 | 사용자 경험 |
|---------|-----------|
| 이미지 생성 실패 (1개) | 해당 우주만 일러스트 fallback + "이 차원은 불안정합니다" 텍스트 |
| 이미지 생성 전체 실패 | "차원 균열이 너무 심합니다. 다시 시도해주세요" + 재시도 버튼 |
| API 타임아웃 (30초) | 위 fallback 적용 |
| 얼굴 미검출 | 업로드 즉시 안내, 다시 업로드 유도 (생성 시작 전 체크) |
| 부적절 이미지 | Nano Banana의 safety filter에 위임 |
| 저사양 모바일 | `prefers-reduced-motion` 감지 시 글리치/포탈 → 단순 페이드로 fallback |

## Design System (plan-design-review에서 확정)

### Color Palette
- `--bg`: #0a0a1a (딥 네이비)
- `--surface`: #12121f
- `--border`: #1f1f35
- `--text-primary`: #e8e8ed
- `--text-secondary`: #8888a0
- `--accent`: #f59e0b (앰버, 차원 균열의 빛)
- `--accent-glow`: #f59e0b33
- `--error`: #ef4444
- `--success`: #22c55e

### Universe Tier Colors
- Tier 1 (현실): warm neutral (베이지, 세피아)
- Tier 2 (판타지): saturated (진홍, 에메랄드, 골드)
- Tier 3 (기상천외): neon/이질적 (핫핑크, 전기블루, 형광)

### Typography
- UI/본문: Pretendard (400, 600, 700)
- 우주 이름 + 스토리: Noto Serif KR (400, 700)
- 글리치 텍스트: monospace (tracking wide)
- Title: 28px / bold / tight
- Subtitle: 14px / regular / --text-secondary
- Body: 15px / regular / line-height 1.7
- Caption: 12px / regular

### Spacing Scale
xs: 4px, sm: 8px, md: 16px, lg: 24px, xl: 40px, 2xl: 60px

### Border Radius
- Card: 16px, Button: 12px, Input: 12px, Badge: 20px (pill), Photo: 50%

### Motion
- 포탈 오픈: 800ms ease-out
- 화면 분열: 600ms + 200ms delay
- 블랙아웃→리빌: 400ms fade-out + 600ms fade-in
- 글리치: 100ms 간격 랜덤 오프셋, 3-5회
- 타이핑 효과: 30ms/글자
- prefers-reduced-motion: 글리치/포탈 → 단순 페이드, 타이핑 → 즉시 표시

### Information Architecture (화면별 위계)

**Landing**: 제목 → 사진 업로드 → 이름 입력 → CTA → 개인정보 안내
**Verse Jump**: 질문 → 3개 선택지 → 진행 인디케이터
**Loading**: 차원 균열 텍스트 (중앙, 큰 글씨) → 진행 표시
**Universe Reveal**: AI 이미지 (상단 60%) → 우주 이름 → 3줄 스토리 (타이핑) → 스와이프 힌트
**Results Grid**: 타이틀 + 진행도 → 5개 그리드 → "다시 점프" → 공유 버튼 → CTA
**/result/[id]** (공유): 대표 이미지 1장 + 나머지 4개 블러 + "나도 해보기" CTA

### Accessibility
- 키보드: Tab+Enter (선택지), 화살표 키 (스와이프 대체)
- 터치 타겟: 최소 44px
- 색상 대비: AAA 통과 (15.5:1, 8.3:1)
- prefers-reduced-motion 대응
- 이미지 alt: "[우주이름] - [테마]"
- 스크린 리더: "Universe 1 of 5"

### Responsive
- 390px (기본): 단일 컬럼, 풀 너비
- 768px: 좌우 패딩 증가, Results Grid 2×3
- 1024px+: max-width 480px 컨테이너, 중앙 정렬

## What I noticed about how you think

- "둘 다"를 세 번 연속 선택했어요. 이미지 생성이냐 스토리냐, URL이냐 이미지 카드냐... 타협하지 않고 풀 경험을 추구하는 성향. 결과물의 완성도에 대한 기준이 높습니다.
- "영화에서는 그게 다른 평행세계를 건너는 방법이거든"이라고 했을 때, 단순히 UI를 말한 게 아니라 영화의 핵심 메카닉을 웹 경험으로 치환하는 사고를 하고 있었어요. "verse jumping"의 본질을 웹 인터랙션으로 번역한 겁니다.
- 10x 버전으로 "시네마틱 연출"을 골랐어요. 기능이 아니라 감각을 고른 거예요. 이 프로젝트의 가치가 "뭘 보여주냐"가 아니라 "어떻게 느끼게 하냐"에 있다는 걸 직감적으로 알고 있습니다.
- Nano Banana를 직접 써보고 "얼굴 유지가 어느 정도 되더라고"라고 한 것. 이론이 아니라 실제 테스트를 거쳐서 도구를 추천한 거예요. 빌더의 본능입니다.
